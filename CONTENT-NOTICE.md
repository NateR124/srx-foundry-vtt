# Notice regarding the bundled SRX content

**This is a notice, not a license.** It does not grant you anything. It exists
to tell you accurately whose work the compendium content is, on what terms it
got here, and who to ask if you want to use it yourself.

## What this covers

`packs/` and `packs-src/` — roughly 1,600 documents covering SRX weapons,
armor, gear, augments, spells, magic gear, talents, traits, contacts, pregen
characters, and threats, along with their stats and short mechanical
descriptions.

It does not cover `packs-src/srx-guides/`, which is original onboarding
material written for this project and falls under the code license (see
`LICENSE`).

## How it got here

RampantGames was asked directly, in the SRX community Discord, whether he
objected to a Foundry VTT system for SRX being built — including explicit
disclosure that it would be built with AI assistance. On 2026-07-11 he replied:

> don't think so! if it helps people get playing, that seems worthwhile
>
> (and as long as you're not charging anyone actual money for it)

and on 2026-07-13, once it was clear the project was a full game system rather
than an add-on module:

> then yes, would be awesome to have SRX on Foundry!

That is the permission this project operates under. It is genuine and it came
from the owner, but it is informal, and its edges were never negotiated.

## The condition

The permission came with one condition, stated plainly: **nobody charges money
for it.** This project honors that, and asks that you do too.

This is why the code in this repository is released under a permissive open
source license but the content is not. Open source licenses — MIT, 0BSD, and
every other OSI-approved license — permit commercial use by design. Applying
one to this content would hand out a right that was never given.

## What was never settled

Two things were never actually discussed, and this notice will not pretend
otherwise:

- **Scope.** The conversation was about building a Foundry system for SRX. It
  did not specifically cover bundling the full catalog as data. The project
  author believes this is within the spirit of what was agreed and intends to
  confirm it; until that confirmation is recorded here, treat it as an
  assumption rather than a fact.
- **Whether it travels.** The permission was given to this project. Nothing was
  said about whether it extends to anyone who forks this repository. Assume it
  does not.

## So if you want to use this content

Ask RampantGames. Your permission comes from him, not from this repository, and
he is by all evidence a reasonable person to ask. Forking the code is a
different matter — that's 0BSD, help yourself.

If RampantGames wants any of this changed, removed, or relicensed, his call
wins over anything written here, and this notice will be updated to say
whatever he wants it to say.
